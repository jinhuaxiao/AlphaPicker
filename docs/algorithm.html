Decision Algorithm Spec

系统中并存两套算法：① 主算法 — 机会决策算法 v2（机会决策页 / 智能推荐使用）；② 遗留 — 五维加权综合分（评分卡 / 仪表盘的「综合分」）。下文以真实代码常量为准（src/lib/alpha.ts、scoring.ts、economics.ts）。

一 · 主算法：机会决策算法 v2 src/lib/alpha.ts

① 卖家策略→ ② TACOS 财务底座→ ③ 市场机会分→ ④ 利润/风险/适配 三乘数→ ⑤ VOC 人机确认→ ⑥ 机会指数→ ⑦ 决策 + 动态预案

核心原则：不要让 AI 做客观判断、不要用伪精确分数包装不确定性 —— 用硬性门槛 + 真实利润底座 + 市场结构 + 人机确认闭环 + 竞品预案替代单一「评分 78 分」。

① 卖家策略 buildSellerPolicy(seller)

risk = risk_preference / 100（0 保守 ~ 1 激进）；按经验取基线，再被风险偏好微调。

② TACOS 财务底座 calculateBlendedFinanceWithTacos(e, policy)

售前毛利率  grossMargin = (price − (price·15%佣金 + FBA) − 到岸成本) / price   // 由真实/估算 COGS 反推
广告订单占比 paidShare   = clamp(0.3 + top3集中度/200, 0.3, 0.7)
预估 TACOS  estTacos     = estACOS × paidShare                 // 总广告成本占比（含自然单）
可承受 TACOS maxTacos    = grossMargin − 目标净利率
TACOS 安全边际          = maxTacos − estTacos
TACOS 后净利率          = grossMargin − estTacos
财务硬门槛 passesFinanceGate = (TACOS后净利率 > 0)

单位经济来自 computePnl()：单件净贡献 = 净售价 − Amazon费 − 广告费 − 到岸成本；并给出盈亏临界 ACOS、回本周期、ROI = 月净利 / 备货资金。

③ 市场机会 analyzeMarketOpportunity(e, policy)

需求分   demand = clamp( log10(月搜索) / log10(50000) × 100 )
长尾空间        = 100 − top3集中度
竞争分   conc   = clamp( 100 − (top3 − 20) × 1.25 )
市场机会分 marketScore = demand×0.5 + conc×0.3 + 长尾×0.2
需求硬门槛 passesDemandGate = (月搜索 ≥ minMonthlySearch)

④⑤ VOC 痛点 · 人机确认闭环

extractVocPainPointsWithAI() 按品类聚类出痛点（严重度 高 中 低，初始 supplierSolvable = null）；collectSellerConfirmationForVoc() 合并卖家「供应商可解决 / 无法解决」确认，直接进入 VOC 系数。（当前为启发式 stub，可替换为真实 LLM 对评论做 VOC 抽取。）

⑥ 机会指数 calculateOpportunityIndex

硬性门槛（三项全过才 gatePass）： 需求门槛 ∧ 财务门槛 ∧ top3 ≤ 80%。

利润乘数 = clamp(0.72 + TACOS后净利率/60, 0.72, 1.08)
风险乘数 = clamp( (0.74 + TACOS安全/45 + (50−top3)/180) × (0.92 + risk×0.16), 0.65, 1.08 )
卖家适配 = clamp( (类目命中?1.05:0.95) × 预算适配, 0.75, 1.10 )    // 预算适配 = 资金需求≤预算?1:预算/资金需求
VOC 系数 = clamp( 1 − 未确认高危×0.06 − 无法解决×0.12, 0.55, 1 )

structural = clamp( marketScore × 利润 × 风险 × 适配, 0, 100 )      // 先夹到 100
机会指数   = structural × VOC系数                                // 再乘 VOC，让人机确认真实拉动分数
若任一硬门槛未过 → 机会指数 = min(机会指数, 38)

⑦ 决策与动态预案 generateDecisionAndWarnings

条件决策等级
任一硬门槛未过不建议进入 avoid
机会指数 ≥ 75建议进入·可正常仓位 enter_and_scale
62 ≤ 指数 < 75建议进入·标准测试 enter
45 ≤ 指数 < 62观望 / 小批量测试 observe_or_micro_test
指数 < 45不建议进入 avoid

同时输出：决策理由 reasoning、预算策略（初始预算 = 预算×测试占比；首批数量按等级 50–500 units）、关键词策略（按 top3 集中度切长尾或主词）、止损规则（TACOS 超 max(8, 可承受TACOS)、退货率>8%、核心词 CPC +20%、前 10 评论<4.0★）、竞品动态预案（竞品上 20% coupon、CPC +20% 等触发→影响→应对）。

二 · 遗留算法：五维加权综合分 src/lib/scoring.ts

仍计算并存入 composite，显示在评分卡 / 仪表盘的「综合分」。

五维（deriveScores，0–100）

  • 需求规模 ← 月搜索（log 缩放）
  • 竞争 ← Top3 集中度（反转，越分散越高）
  • 利润空间 ← 毛利率
  • 差异化 ← 未填补卖点数
  • 风险 ← 退货率 + ACOS 安全边际（反转）

综合分 = Σ(维度分 × 权重) / Σ权重
状态：综合分≥60 且 风险≥40 → 推荐；综合分<45 或 风险<30 → 不建议；否则 观望
数据来源：当前实时数据源为 Sorftime MCP（Amazon）与 eBay Browse API（互为备份）。毛利率由真实/估算 COGS 反推，使「利润评分」与「盈亏模拟」共用同一成本基准、结论一致。VOC 抽取当前为启发式占位，可接入真实 LLM。
